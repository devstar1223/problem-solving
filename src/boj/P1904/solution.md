# 🔍 문제 해석

- 다음 두 타일이 무한히 많이 있다.
  - `00` 으로 구성된 두장의 타일
  - `1` 로 구성된 낱장 타일
- N이 주어졌을때, **크기(길이)가 N인 두개의 타일로 만들 수 있는 2진 수열**의 갯수를 구하라.
  - N은 1 ~ 100만 이다.
- 15746으로 나눈 나머지를 출력하라.
***

# 🛠️ 풀이

- N개만큼 빈 공간이 있고, 이곳에는 0 또는 1을 넣을수 있다고 생각해본다.
  - 0과 1의 타일만 있다면 2^N 만큼 넣을 수 있다.
- 5개의 칸에 왼쪽부터 0 또는 1을 채울수 있는지 없는지를 생각해본다. (왼쪽이든 오른쪽이든 크게 상관은 없다.)
  - ㅁ ㅁ ㅁ ㅁ ㅁ
  - 왼쪽에 `0`을 넣을경우 (0 ㅁ ㅁ ㅁ ㅁ)
    - 남은 네개의 칸에 자유롭게 숫자를 넣을 수 있다.
      - **즉, 처음부터 4개의 칸이 있을때 가능한 갯수를 이미 알고있다면 다시 계산할 필요가 없다.**
  - 왼쪽에 `11`을 넣을경우 (1 1 ㅁ ㅁ ㅁ)
    - 남은 세개의 칸에 자유롭게 숫자를 넣을 수 있다.
      - **위와 같이, 이미 알고있다면 다시 계산할 필요가 없다.**
- 어느정도 규칙을 찾았으니, 3개의 칸부터 가정해본다.
  - 1개의 칸은 `0` 1개만 가능하고, 2개의 칸은 `00`과 `11` 2개만 가능한걸 알고있다.
  - 왼쪽에 `0` 삽입 (0 ㅁ ㅁ)
    - 남은 칸은 총합 2개의 조합이 가능하다 (=2개의 칸의 갯수)
  - 왼쪽에 `11` 삽입 (1 1 ㅁ)
    - 남은 칸은 총합 1개의 조합이 가능하다 (=1개의 칸의 갯수)
- N개의 칸이 있을때
  - 왼쪽에 `0` 삽입 (0 ㅁ ㅁ ㅁ ㅁ ㅁ ㅁ ... ㅁ)
    - 남은 칸은 N-1개의 칸의 갯수만큼 가능하다.
  - 왼쪽에 `11` 삽입 (1 1 ㅁ ㅁ ㅁ ㅁ ㅁ ... ㅁ)
    - 남은 칸은 N-2개의 칸의 갯수만큼 가능하다.
- N = (N-1) + (N-2)
  - 남은건 탑다운 점화식을 코드로 구현하는것 뿐이다.
  - 담을때마다 15746 으로 나눠서 나머지만 담기.
***
- 문제1 : 스택오버플로우
  - 결국 `30000` 정도의 값을 넣게되면, 매우 많이 호출을 하게 되고... 스택오버플로우가 터진다.
  - 아예 3부터 N까지 바텀업으로 천천히 구하는건 어떨까. N이 1백만 정도로 아주 크지는 않으니까 괜찮지않을까.

# ⏱️ 시간 복잡도

시간 복잡도 : $O(N)$
