# 🔍 문제 해석

- 1*N 크기의 미로가 있다. (세로1 가로N, N은 최대 1000)
- 각 칸에는 정수가 하나씩 쓰여져있다. (최대 100)
    - 각 칸마다 쓰여있는 수 만큼 오른쪽으로 이동 가능하다.
    - ex) 특정 칸에 3이 쓰여 있다면, +1/+2/+3 중 선택하여 이동 가능
- 재환이는 미로의 가장 왼쪽 끝에 있다.
- 오른쪽에 가기 위한 최소 점프 수는?
  - 갈 수 없다면 -1을 출력한다.

***

# 🛠️ 풀이

- 숫자 0이 쓰여져있는 칸에 도착할 수 밖에 없다면, 무조건 -1이니 주의하자.
- 2번째 ~ N번째 칸 까지 각 칸마다, 해당 칸에 도달하는 최소 점프 횟수를 `int[] answer` 기록해두면 된다.
  - 맨 왼쪽칸부터 시작하여, 현재 점프 수 에서 갈수 있는 칸에 `answer` 의 값이 있는지 확인하고, 없으면 기록한다. (편의상 첫 칸을 제외하고 `-1`로 둔다.)
    - 값이 이미 있다면 기록할 필요가 없다. (이미 더 빠른 방법으로 도달 가능.)
  - `answer[N-1]`에 적힌 수가 정답이다.
***
- 제출 결과 틀렸다. 간과한 부분이 무엇일까?
  - 정답칸이 3칸뒤인데, 현재 칸이 5일때?
    - 현재 칸에서 +1 +2 +3 +4 +5인 지점들을 차례대로 점프하고, 배열보다 커질 경우도 예외로 두었기 때문에 문제가 되지 않는다.
  -  -1 출력 부분이 문제인듯 싶어, 아래 값을 넣어보니 틀린 수가 나왔다.
    - ```
      6
      3 0 0 0 1 1
      
      올바른 답 : -1
      출력값 : 0
      ```
  - **그 칸에 뭐가 쓰여있던 간에, 무조건 왼쪽칸부터 오른쪽칸까지 계속 뛸 수 있게 for문을 돌린게 잘못이었다.**
  - 무조건 뛰게 하는게 아니라, `answer[i]`의 값이 -1이 아닌 값만 뛰어야 한다.
    - 따라서, 현재 칸의 `jumpAble`만큼 각 칸마다 점프수를 더해주기 전에, 현재 칸의 최소 도달 횟수 `answer[i]`가 `-1`일 경우 계산하지않고 넘어간다.

***

# ⏱️ 시간 복잡도

시간 복잡도 : $O(N)$
- 가장 바깥의 루프인 `for(int i = 0; i < N; i++)` 부분에서, N은 최대 1000만큼 가능하다.
- 그 안의 루프인 `for(int j = 1; j <= jumpAble; j++)` 부분에서, `jumbAble`은 최대 100만큼 가능하다.
- N * 100 -> N

# ☕ 여담

BFS로도 풀이가 가능하다고 한다.
뭐 하긴 미로문제(= 이미 가본 곳은 안가도 되는)가 늘 그렇지.

## 실수 분석
### 원인
왼쪽부터 끝까지 순회하면서, 현재 칸에서 갈 수 있는곳들을 갱신하면 편하겠다고 생각해서, 이렇게 상태를 전제했다.
> 각 칸을 현재 위치라고 생각하고 계속해서 점프만큼만 더해주면 된다. 이미 가본칸만 패스하면 문제없다.

**하지만 실제로는 도달할 수 없는 칸도 존재하며, 도달할 수 없는 칸에서도 점프해서 결과값을 갱신하고 있었다.**

### 해결 방안
> 현재 칸에서 점프를 시도할 수 있는 조건은, 도달 가능한 칸만 가능하다. 도달할 수 없는 칸에서는 점프하지 않는다.
